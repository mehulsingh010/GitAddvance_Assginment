Question 1 : What happens when a file is deleted in the main branch and updated in a feature branch, and how is this handled during a merge?

Answer :- When a file is deleted in the main branch and updated in a feature branch, Git sees this as a conflict during the merge.


In the main branch, the file is deleted.
In the feature branch, the same file is updated.
During the merge, Git tries to combine the changes. Since the file is deleted in main and modified in feature, Git will keep the version from the feature branch and restore the file with the updated content. This is because Git assumes the changes in the feature branch are more important than the deletion in the main branch.

We'll need to resolve this conflict manually if Git can't decide, and after resolving, we must commit the changes to finalize the merge.

$ git add conflict.txt
 
Mehul@hppavilion MINGW64 /d/GitAdvance_Assg_Personal (main)
