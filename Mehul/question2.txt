Question 2 :- 2. What does the git `ls-files` command do, and what information does its output provide regarding the state of files in a Git repository?

Answer : The `git ls-files` command used to lists the files in the staging area.

It basically Shows which files are tracked by Git and their status.

Options:
`-o` (others): Lists untracked files.
`-d` (deleted): Lists deleted files.
`-m` (modified): Lists modified files.

Demo : 

Mehul@hppavilion MINGW64 /d/GitAdvance_Assg_Personal (main)
$ ls
conflict.txt  one/  three/  two/

Mehul@hppavilion MINGW64 /d/GitAdvance_Assg_Personal (main)
$ cd one

Mehul@hppavilion MINGW64 /d/GitAdvance_Assg_Personal/one (main)
$ ls
f1.txt  f2.txt

Mehul@hppavilion MINGW64 /d/GitAdvance_Assg_Personal/one (main)
$ git ls-files
f1.txt
f2.txt

Mehul@hppavilion MINGW64 /d/GitAdvance_Assg_Personal/one (main)
$ echo "new untracked file" >> untrackedfile.txt

Mehul@hppavilion MINGW64 /d/GitAdvance_Assg_Personal/one (main)
$ git ls-files
f1.txt
f2.txt

Mehul@hppavilion MINGW64 /d/GitAdvance_Assg_Personal/one (main)
$ git ls-files -o
untrackedfile.txt

Mehul@hppavilion MINGW64 /d/GitAdvance_Assg_Personal/one (main)
$ echo "delete file" >> deletefile.txt

Mehul@hppavilion MINGW64 /d/GitAdvance_Assg_Personal/one (main)
